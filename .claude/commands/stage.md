Git stage les hunks concernés par le chantier en cours (et uniquement ceux-là), puis propose un message de commit au format conventionnel du projet (voir CLAUDE.md). Ne lance pas le commit — attends ma validation.

Règles pour le message :

- Rédige-le en anglais.
- Si une description longue est utile, liste les points sous forme de tirets (`- ...`).
- Si tu finis par créer le commit toi-même, n'ajoute AUCUNE mention de co-auteur (pas de `Co-Authored-By: Claude ...`).
